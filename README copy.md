# Sensory Journey

> *A top-down pixel-art game about navigating the world with sensory overload.*  
> Visual style inspired by **Pokémon Ruby** (Game Boy Advance).

---

## 🎮 Concept

You play as **Alex**, a character who experiences sensory processing differences consistent with autism.  
The game follows one ordinary errand — going to the grocery store and coming home — across three levels.

| Level | Location | Challenge |
|-------|----------|-----------|
| 1 | **Home** | Notice the fridge is empty and prepare to go out |
| 2 | **The Street** | Cross a busy road to reach the grocery store |
| 3 | **The Park** | The direct route home is blocked; cut through the park |

Throughout each level a **Sensory Load bar** fills as you encounter noise sources. If it maxes out, you are sent back to the start of the level. Quiet zones drain it. The world doesn't slow down — you have to manage it.

---

## 🕹️ Controls

| Key | Action |
|-----|--------|
| **WASD** or **Arrow Keys** | Move |
| **SHIFT** (hold) | Focus Mode — slows you down slightly and shows a dotted guide path to your next target |
| **ENTER** | Confirm / advance dialogue |
| **Z** or **Space** | Also advances dialogue |
| **R** | Restart current level (or restart from Level 1 on the ending screen) |

## ⚙️ Setup & Running

### Option A — Local (recommended for submission)

1. **Download p5.js** from [https://p5js.org/download/](https://p5js.org/download/)  
   → Click **"p5.js complete"** → save `p5.min.js`  
   → Place it in `libraries/p5.min.js`

2. **Start a local server** (required — browsers block local file loading):
   - VS Code: install the **Live Server** extension → right-click `index.html` → *Open with Live Server*  
   - Or run in terminal: `npx serve .` then open `http://localhost:3000`

3. Open `index.html` in a browser.  
   The game will load automatically. No build step required.

> **Note:** If `libraries/p5.min.js` is missing, the HTML automatically falls back to the CDN version — so the game still runs as long as you have internet access.

### Option B — GitHub Pages

1. Push the entire folder to a GitHub repository.
2. In Settings → Pages → select your branch → save.
3. The CDN fallback in `index.html` handles p5.js automatically.
4. Visit `https://<username>.github.io/<repo>/`.

## 🎨 Visual Design Notes

- **Tile size:** 32×32 px (pixel art drawn at 16×16, scaled 2× for GBA-style chunky pixels)
- **Canvas:** Fixed 640×480 px — same aspect ratio as the GBA screen
- **Palette:** Hand-curated GBA colours matching Pokémon Ruby's outdoor/indoor/urban environments
- **CRT effect:** CSS scanlines + subtle glow applied to the canvas container
- **No external art assets** — all sprites and tiles are procedurally generated by `gen_sprites2.py`

## 🔊 Audio Notes

The `assets/sounds/` folder is reserved for audio. To add sound:
1. Place `.mp3` or `.wav` files in `assets/sounds/`
2. In `sketch.js` `preload()`, load them: `soundBG = loadSound('assets/sounds/bg.mp3');`
3. Requires the **p5.sound** library — add `libraries/p5.sound.min.js` and a `<script>` tag in `index.html`

Suggested sounds:
- `home_ambient.mp3` — quiet indoor hum
- `street_ambient.mp3` — traffic, crowd noise
- `park_ambient.mp3` — birds, distant activity
- `overload.wav` — sharp distortion burst on sensory max
